Como Forçar 100% de GPU no Ollama (Modelfile)

Dica de otimização local para LLMs multimodais em placas de 8 GB de VRAM

Ao carregar modelos de 12B com capacidades de visão (como o Gemma 4) em GPUs de 8 GB, o Ollama frequentemente divide o processamento com a CPU (ex: 32% CPU / 68% GPU). Isso ocorre porque o tamanho do modelo somado ao cache de contexto estoura os limites físicos da VRAM, gerando lentidão extrema no processamento de imagens.

A solução definitiva envolve baixar o modelo correto e criar uma versão customizada utilizando um Modelfile para congelar os limites de memória e proibir o uso da CPU.

Passo a Passo

  1. Baixe o modelo base da Unsloth:

    Execute o comando abaixo no terminal para puxar o arquivo quantizado diretamente do Hugging Face. O Ollama fará o download automático dos mais de 7 GB e iniciará o chat assim que concluir:

    ollama run hf.co/unsloth/gemma-4-12b-it-GGUF:UD-Q4_K_XL
  2. Crie o arquivo de configuração (Modelfile):

    Feche o chat anterior e, na mesma pasta de sua preferência, execute os comandos abaixo para gerar as instruções de hardware:

    echo FROM hf.co/unsloth/gemma-4-12b-it-GGUF:UD-Q4_K_XL > Modelfile
    echo PARAMETER num_ctx 4096 >> Modelfile
    echo PARAMETER num_gpu 99 >> Modelfile
    Entendendo os parâmetros:
    • num_ctx 4096: Reduz a janela de contexto para 4k. Janelas maiores criam um cache de memória gigante que força o transbordo para a CPU ao ler imagens.
    • num_gpu 99: Força o Ollama a enviar todas as camadas possíveis do modelo para a placa de vídeo, bloqueando a divisão de processamento.
  3. Compile o novo modelo otimizado:

    Gere uma nova tag no repositório local do seu Ollama apontando para o arquivo criado:

    ollama create gemma4-gpu -f ./Modelfile
  4. Execute o modelo 100% na GPU:

    Pronto! Agora basta iniciar o novo modelo customizado. A leitura de prints e imagens passará a ser praticamente instantânea:

    ollama run gemma4-gpu

Como Validar

Se quiser verificar a alocação de hardware enquanto faz uma pergunta ou envia um print, abra um segundo terminal em paralelo e digite:

ollama ps

A coluna PROCESSOR deve exibir obrigatoriamente 100% GPU.
