Como forçar 100% de GPU no Ollama

Publicado em 23/07/2026 · #InteligenciaArtificial #Hardware #Ollama

Ao carregar modelos de 12B com capacidades de visão (como o Gemma 4) em placas de vídeo de 8 GB, o Ollama frequentemente divide o processamento com a CPU (ex: 32% CPU / 68% GPU). Isso acontece porque o modelo somado ao cache de contexto estoura a VRAM, deixando a leitura de imagens extremamente lenta. Dá para resolver isso limitando o contexto e proibindo o uso da CPU via Modelfile.

1. Baixar o modelo base otimizado

O primeiro passo é baixar uma versão quantizada estável. Execute o comando abaixo no terminal para puxar o arquivo diretamente do Hugging Face. O Ollama fará o download de aproximadamente 7 GB e iniciará o chat automaticamente:

ollama run hf.co/unsloth/gemma-4-12b-it-GGUF:UD-Q4_K_XL

Assim que o download terminar, feche esse chat inicial no terminal para liberar a memória para os próximos passos.

2. Criar as regras de hardware (Modelfile)

Abra o terminal na pasta onde deseja salvar suas configurações e execute os três comandos abaixo em sequência para gerar o arquivo Modelfile:

echo FROM hf.co/unsloth/gemma-4-12b-it-GGUF:UD-Q4_K_XL > Modelfile
 echo PARAMETER num_ctx 4096 >> Modelfile
 echo PARAMETER num_gpu 99 >> Modelfile

O parâmetro num_ctx 4096 impede que a janela de contexto cresça demais e force o transbordo para a memória RAM comum, enquanto o num_gpu 99 obriga o Ollama a enviar todas as camadas do modelo para a placa de vídeo.

3. Compilar e rodar o modelo customizado

Com o arquivo criado, gere a nova versão otimizada no repositório local do seu Ollama:

ollama create gemma4-gpu -f ./Modelfile

Agora, para iniciar o modelo travado permanentemente em sua placa de vídeo com respostas instantâneas, basta rodar:

ollama run gemma4-gpu

Como validar

Para ter certeza de que funcionou, abra um segundo terminal enquanto faz uma pergunta com imagem e digite ollama ps. A coluna do processador deverá exibir obrigatoriamente 100% GPU.
